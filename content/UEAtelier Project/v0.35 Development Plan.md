# v0.35 Development Plan

Languages: [[v0.35 Development Plan]] · [[v0.35 Development Plan CN]] · [[v0.35 Development Plan JP]]

v0.35 is the transition release that makes UE 5.7 and UE 5.8 the primary support targets and turns the local RAG layer into a dependable product subsystem. **Batch 1 SHIPPED in v0.35.0 (2026-07-14)**: UE 5.7/5.8 primary source targets plus the knowledge-index reliability and retrieval-quality overhaul (release notes `Docs/Release-2026-07b.md`, development contract `Docs/Development-0.35.md`). The dual-variant structure and the optional Epic official-MCP integration were intentionally not part of Batch 1 — they remain the outstanding Batch 2 (see the final section).

Epic's UE 5.8 official MCP stack remains optional and default-off; it must not bypass UEAtelier's current policy, vetting, and audit boundaries.

## Priority 0 — Make the Knowledge Index Trustworthy

SHIPPED in v0.35.0.

1. Gate D and Knowledge eval fixtures use isolated `indexRoot` directories, so test cleanup cannot erase the user's index.
2. Search/recommend/refresh report explicit machine-readable `missing | empty | stale | ready | corrupt` state; Chat refreshes from that field and stops retrying an automatic refresh after the first failure in a panel session.
3. `cards.jsonl`/`index.json` are staged, verified, and replaced through recoverable last-known-good `.bak` pairs; a zero-card refresh fails closed by default, and an interrupted write auto-recovers on the next load. `allowEmptyIndex` is hard-gated to explicit `indexRoot` values under `Saved/UnrealMcp/Tests`. Unchanged warm reads use a size/timestamp fast path.
4. ToolRegistry schemas expose `includeActivityLog`, `includeSkills`, `sourceKinds`, `groupByKind`, plus the new `includePromotedSources` and `allowEmptyIndex` controls.
5. Index schema `UEvolve.KnowledgeIndex.v2` records `buildId`, `generatedAtUtc`, `sourceFingerprint`, `cardsSha256`, card/source/engine counts, deduplication/truncation counts, and build parameters.
6. ActivityLog indexing is explicit opt-in (defaults to false); local-only storage does not by itself authorize injecting chat summaries into provider context. Index containment is fail-closed and does not follow symlinks or reparse points.

## Priority 1 — Improve Retrieval Quality

SHIPPED in v0.35.0.

- Engine-version tokens such as `5.7`, `5.8`, `UE5.7`, and `UE5.8` survive tokenization; plain `x.y` tokens filter only for known engine versions, so product `0.35` or Python `3.11` queries retain versioned cards.
- Latin word boundaries are ASCII-defined and deterministic, so `ui` no longer matches `build` and stays searchable inside CJK prose; CJK bigrams behave identically on every platform.
- Original query tokens rank above synonym expansions (expansions carry a 0.55 weight) instead of both having equal influence.
- Global truncation reserves source-kind/engine diversity before the card cap, so high-weight documentation cannot starve skills, official docs, or bounded runtime evidence; equal-score ordering is deterministic.
- Card metadata separates `sourceUpdatedAt`, `indexedAt`, and `contentSha256` rather than treating every refresh time as the source update time; official-doc rows and KnowledgeCards carry `engineVersion`.
- Activity, promoted Task Atlas, and skill sources stay bounded and chunked with local-only privacy defaults and explicit opt-in for ActivityLog. (Boundary-aware scoring for the separate `ScoreToolForTask` tool-recommendation matcher is a Batch 2 fast-follow.)

## Priority 2 — Turn Evals into Release Gates

SHIPPED in v0.35.0.

- Eval schema v2 (`UEvolve.KnowledgeEval.v2`) supports rank-aware assertions: `expectSourcePathsAtK`, `expectAnySourcePathContains`, `forbidTopSourcePathContains`, and `expectToolAtRank`, with aggregate rank-assertion counts/rate.
- The 17-test RAG automation baseline covers interruption recovery, stale/corrupt append refusal, isolated-empty gating, known-engine numeric filtering, deterministic CJK/Latin retrieval, and original-token rank coverage on both primary engines.
- Privacy and registry-schema parity checks stay in the same gate.
- Run `unreal.knowledge_eval_run` after every ranking, source, or recommendation change (still standing policy; live UE 5.8 run: 8/8 cases, 3/3 rank assertions on an isolated index).
