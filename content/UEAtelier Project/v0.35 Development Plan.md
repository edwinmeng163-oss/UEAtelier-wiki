# v0.35 Development Plan

v0.35 is the transition release that makes UE 5.7 and UE 5.8 the primary support targets and turns the local RAG layer into a dependable product subsystem. The currently released v0.34.0 remains a UE 5.6 + UE 5.7 release; nothing on this page should be read as already shipped.

## Support Decision

| Engine | v0.35 intent |
| --- | --- |
| UE 5.8 | Primary target; required build, automation, endpoint, and package verification |
| UE 5.7 | Primary target; required build, automation, endpoint, and package verification |
| UE 5.6 | Transitional maintenance canary; keep the compile-time floor initially, but do not let it constrain new 5.7/5.8 work |

The existing UE 5.7 sample host will also be used as the UE 5.8 build/commandlet host. Do not add a second multi-gigabyte 5.8 content host. The experimental v0.33 branch is evidence, not a merge base: only the small, reviewed 5.8 compatibility fixes should move forward. Epic's UE 5.8 official MCP stack remains optional and default-off; it must not bypass UEAtelier's current policy, vetting, and audit boundaries.

## Priority 0 â€” Make the Knowledge Index Trustworthy

1. Isolate RAG tests from the default `Saved/UnrealMcp/KnowledgeIndex` so cleanup cannot erase the user's index.
2. Introduce explicit `missing | empty | stale | ready` index state and rebuild automatically for missing, empty, or stale state.
3. Make index replacement atomic and preserve the previous usable index if refresh fails.
4. Expose the already implemented refresh/search controls in ToolRegistry schemas: `includeActivityLog`, `includeSkills`, `sourceKinds`, and `groupByKind`.
5. Record enough manifest metadata to explain freshness and source coverage without exposing private content.
6. Make ActivityLog indexing explicit opt-in; local-only storage does not by itself authorize injecting chat summaries into provider context.

## Priority 1 â€” Improve Retrieval Quality

- Preserve engine-version tokens such as `5.7`, `5.8`, `UE5.7`, and `UE5.8`.
- Use boundary-aware matching for Latin tokens so short terms such as `ui` do not match unrelated words such as `build`.
- Rank original query tokens above synonym expansions instead of giving both equal influence.
- Add source-kind diversity before the global card cap so high-weight documentation cannot starve skills, official docs, or bounded runtime evidence.
- Use stable source timestamps/fingerprints rather than treating every refresh time as the source update time.
- Bound and chunk activity, promoted Task Atlas, and skill sources; retain local-only privacy defaults and explicit opt-outs.

## Priority 2 â€” Turn Evals into Release Gates

- Add rank-aware assertions for top source/tool, required source paths, and forbidden top results.
- Add negative lexical cases, UE 5.7/5.8 version-specific cases, empty-index recovery, stale-index recovery, and source-filter cases.
- Keep privacy and registry-schema parity checks in the same gate.
- Run `unreal.knowledge_eval_run` after every ranking, source, or recommendation change.

## UE 5.7 / 5.8 Compatibility Batches

1. Forward-port only the reviewed JSON object compatibility shim, its call sites/tests, and the UE 5.8 unreachable-break cleanup.
2. Move the root development host association to UE 5.7; reuse the UE 5.7 sample host under UE 5.8.
3. Extend `project_version_migration` to accept UE 5.8 while marking UE 5.6 as legacy/maintenance during the transition.
4. Update installer, package names, CI/release gates, and focused documentation only after both primary engines pass.
