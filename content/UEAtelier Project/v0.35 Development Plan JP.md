# v0.35 開発計画

言語：[[v0.35 Development Plan]] · [[v0.35 Development Plan CN]] · [[v0.35 Development Plan JP]]

v0.35 は、UE 5.7 と UE 5.8 を優先サポート対象に定め、ローカル RAG を信頼できる製品サブシステムへ引き上げる移行リリースです。現在公開済みの v0.34.0 が保証するのは UE 5.6 + UE 5.7 のままです。このページは計画であり、実装・リリース済みの機能を示すものではありません。

## サポート方針

| Engine | v0.35 の方針 |
| --- | --- |
| UE 5.8 | 最優先。build、automation、endpoint、package 検証を必須化 |
| UE 5.7 | 最優先。build、automation、endpoint、package 検証を必須化 |
| UE 5.6 | 移行期の maintenance canary。初期 batch では compile-time floor を維持するが、5.7/5.8 の新規作業を制約させない |

既存の UE 5.7 sample host を UE 5.8 の build/commandlet host としても使い、数 GB の完全な 5.8 content host は追加しません。v0.33 実験 branch は検証資料であって merge base ではなく、レビュー済みの小さな 5.8 互換修正だけを forward-port します。Epic の UE 5.8 official MCP は optional / default-off のままとし、UEAtelier の policy、vetting、audit 境界を迂回させません。

## P0 — Knowledge Index を信頼可能にする

1. RAG tests を既定の `Saved/UnrealMcp/KnowledgeIndex` から隔離し、cleanup がユーザーの index を消さないようにします。
2. `missing | empty | stale | ready` の明示的な状態を導入し、missing / empty / stale を自動再構築します。
3. index を atomic に置換し、refresh 失敗時は直前の利用可能な index を保持します。
4. 実装済みだが schema に未公開の `includeActivityLog`、`includeSkills`、`sourceKinds`、`groupByKind` を ToolRegistry に公開します。
5. private content を漏らさず、freshness と source coverage を説明できる manifest metadata を記録します。
6. ActivityLog indexing を explicit opt-in にします。local-only storage であることだけでは、chat summary を provider context へ注入する権限にはなりません。

## P1 — Retrieval 品質を改善する

- `5.7`、`5.8`、`UE5.7`、`UE5.8` などの engine-version token を保持します。
- Latin token は境界を認識して照合し、短い `ui` が `build` に誤ヒットする問題を防ぎます。
- original query token を synonym expansion より高く評価します。
- global card cap の前に source-kind diversity を確保し、高 weight docs が skills、official docs、制限された runtime evidence を排除しないようにします。
- refresh 時刻ではなく、安定した source timestamp/fingerprint を使います。
- ActivityLog、Task Atlas promotion、skill source を上限付きで chunk 化し、local-only default と明示的 opt-out を維持します。

## P2 — Eval を Release Gate にする

- top source/tool、必須 source path、禁止 top result を検証できる rank-aware assertion を追加します。
- negative lexical、UE 5.7/5.8 version-specific、empty/stale index recovery、source filter case を追加します。
- privacy と registry-schema parity も同じ gate で検証します。
- ranking、source、recommendation の変更後は必ず `unreal.knowledge_eval_run` を実行します。

## UE 5.7 / 5.8 互換 Batch

1. レビュー済みの JSON object compat shim、call sites/tests、UE 5.8 unreachable-break cleanup だけを forward-port します。
2. root development host を UE 5.7 に変更し、UE 5.8 でも UE 5.7 sample host を再利用します。
3. `project_version_migration` に UE 5.8 を追加し、移行期の UE 5.6 は legacy/maintenance と表示します。
4. 両 primary engine が通過してから installer、package name、CI/release gate、install docs を更新します。

## Release Gates

- UE 5.7 / UE 5.8 clean build と完全な `UnrealMcp.*` automation。
- 両 primary engine で endpoint smoke と MCP SDK conformance。
- clean local state からの RAG index recovery と rank-aware eval。
- ToolRegistry 190/190 validation と plugin mirror の byte-identical 確認。
- v0.34 の vetted tools、`call_tool`、Task Atlas、protocol shape safety baseline を維持。
- macOS / Windows package を両 engine で検証してから、文書上の表現を「planned」から「supported」へ変更。

## 最初の v0.35 Batch で行わないこと

- `experiment/v0.33-ue58-validation` の一括 merge。
- full-content UE 5.8 sample project の commit。
- official-MCP Task Atlas generator または policy bypass。
- UE 5.6 compile compatibility の即時削除。EOL は別途、検証結果に基づいて判断します。
