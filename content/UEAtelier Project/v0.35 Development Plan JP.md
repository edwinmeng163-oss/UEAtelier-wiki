# v0.35 開発計画

言語：[[v0.35 Development Plan]] · [[v0.35 Development Plan CN]] · [[v0.35 Development Plan JP]]

v0.35 は、UE 5.7 と UE 5.8 を優先サポート対象に定め、ローカル RAG を信頼できる製品サブシステムへ引き上げる移行リリースです。**Batch 1 は v0.35.0（2026-07-14）としてリリース済み**：UE 5.7/5.8 のプライマリソースターゲット + 知識インデックスの信頼性・検索品質の全面強化（release notes は `Docs/Release-2026-07b.md`、開発契約は `Docs/Development-0.35.md`）。dual-variant 構造と Epic official-MCP 統合（オプション）は意図的に Batch 1 に含めず、残る Batch 2 として扱います（末尾のセクションを参照）。

## P0 — Knowledge Index を信頼可能にする

v0.35.0 で出荷済み。

1. Gate D と Knowledge eval fixtures は隔離された `indexRoot` ディレクトリを使い、テストの cleanup がユーザーの index を消せません。
2. 検索/推薦/refresh は機械可読の `missing | empty | stale | ready | corrupt` 状態を返します。Chat はこのフィールドで refresh し、パネルセッション内で自動 refresh が一度失敗したら再試行しません。
3. `cards.jsonl`/`index.json` は復旧可能な last-known-good `.bak` ペアを介して staged/verified 置換されます。ゼロカードの refresh は既定で fail-closed、書き込みが中断されても次回ロード時に自動復旧します。`allowEmptyIndex` は `Saved/UnrealMcp/Tests` 配下の明示的 `indexRoot` に限定されます。変更のないウォームリードは size/timestamp の高速パスを使います。
4. ToolRegistry schema は `includeActivityLog`、`includeSkills`、`sourceKinds`、`groupByKind` に加え、新設の `includePromotedSources` と `allowEmptyIndex` を公開します。
5. インデックス schema `UEvolve.KnowledgeIndex.v2` は `buildId`、`generatedAtUtc`、`sourceFingerprint`、`cardsSha256`、カード/ソース/エンジン数、重複排除/切り詰め数、ビルドパラメータを記録します。
6. ActivityLog indexing は explicit opt-in（デフォルト false）です。local-only storage であることだけでは、chat summary を provider context へ注入する権限にはなりません。インデックスの containment は fail-closed で、symlink/reparse point を辿りません。

## P1 — Retrieval 品質を改善する

v0.35.0 で出荷済み。

- `5.7`、`5.8`、`UE5.7`、`UE5.8` などの engine-version token はトークン化を生き残ります。素の `x.y` 数値 token は既知のエンジンバージョンにのみフィルタとして働くため、プロダクト `0.35` や Python `3.11` のクエリがバージョン付きドキュメントを隠しません。
- Latin の語境界は ASCII 定義で決定的です：`ui` は `build` に誤ヒットせず、CJK 文中でも検索可能。CJK bigram は全プラットフォームで同一挙動です。
- original query token は synonym expansion（重み 0.55）より上位にランクされ、等価な影響ではなくなりました。
- グローバル切り詰めは card cap の前に source-kind/エンジン多様性を予約し、高 weight docs が skills、official docs、制限された runtime evidence を排除できません。同スコアの順序は決定的です。
- カードのメタデータは `sourceUpdatedAt`、`indexedAt`、`contentSha256` を分離し、refresh 時刻をソース更新時刻として扱いません。official-doc 行と KnowledgeCard は `engineVersion` を持ちます。
- ActivityLog、Task Atlas promotion、skill source は上限付き・chunk 化のまま、local-only default と ActivityLog の explicit opt-in を維持します。（別系統の `ScoreToolForTask` ツール推薦マッチャーの境界認識スコアリングは Batch 2 の fast-follow です。）

## P2 — Eval を Release Gate にする

v0.35.0 で出荷済み。

- Eval schema v2（`UEvolve.KnowledgeEval.v2`）は rank-aware assertion をサポートします：`expectSourcePathsAtK`、`expectAnySourcePathContains`、`forbidTopSourcePathContains`、`expectToolAtRank`。rank-assertion の件数/通過率も集計します。
- 17 テストの RAG automation ベースラインは、中断復旧、stale/corrupt append 拒否、隔離空インデックスのゲート、既知エンジンの数値フィルタ、決定的な CJK/Latin 検索、original-token ランクを両プライマリエンジンでカバーします。
- privacy と registry-schema parity のチェックは同じ gate に残ります。
- ranking、source、recommendation の変更後は必ず `unreal.knowledge_eval_run` を実行します（恒久ルール。UE 5.8 実機実行：隔離インデックスで 8/8 case、3/3 rank assertion）。
