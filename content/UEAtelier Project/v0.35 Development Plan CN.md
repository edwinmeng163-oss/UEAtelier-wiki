# v0.35 开发计划

语言：[[v0.35 Development Plan]] · [[v0.35 Development Plan CN]] · [[v0.35 Development Plan JP]]

v0.35 是一次过渡版本：将 UE 5.7 与 UE 5.8 确立为优先支持目标，并把本地 RAG 从“可用功能”提升为可靠的产品子系统。当前已发布的 v0.34.0 仍然只承诺 UE 5.6 + UE 5.7；本页内容均为计划，不能描述成已经发布。

## 支持决策

| 引擎 | v0.35 目标 |
| --- | --- |
| UE 5.8 | 第一优先；必须通过构建、自动化、endpoint 与发布包验证 |
| UE 5.7 | 第一优先；必须通过构建、自动化、endpoint 与发布包验证 |
| UE 5.6 | 过渡期 maintenance canary；首批暂不抬高编译下限，但不再让 5.6 限制 5.7/5.8 的新工作 |

继续复用现有 UE 5.7 sample host 作为 UE 5.8 的构建/commandlet host，不新增数 GB 的完整 5.8 内容工程。v0.33 实验分支只作为验证证据，不作为合并基线；仅前移经过审查的最小 5.8 兼容补丁。Epic 的 UE 5.8 official MCP 保持可选、默认关闭，并且不得绕过 UEAtelier 现有的 policy、vetting 与 audit 边界。

## P0 — 先让知识索引可信

1. 将 RAG 测试与默认 `Saved/UnrealMcp/KnowledgeIndex` 隔离，测试清理绝不能抹掉用户索引。
2. 引入明确的 `missing | empty | stale | ready` 状态；对 missing、empty、stale 自动重建。
3. 原子替换索引；refresh 失败时保留上一份可用索引。
4. 把代码里已经存在、但 schema 未公开的参数补进 ToolRegistry：`includeActivityLog`、`includeSkills`、`sourceKinds`、`groupByKind`。
5. 在 manifest 中记录足够的 freshness 与 source coverage 元数据，同时不泄露私有内容。
6. 将 ActivityLog 索引改为显式 opt-in；仅仅是 local-only 存储，不等于已经授权把 chat summary 注入 provider context。

## P1 — 提升检索质量

- 保留 `5.7`、`5.8`、`UE5.7`、`UE5.8` 等引擎版本 token。
- Latin token 使用边界匹配，避免短词 `ui` 错误命中 `build`。
- 原始 query token 权重高于 synonym expansion，不再等权。
- 在全局卡片上限之前保证 source-kind 多样性，避免高权重文档挤掉 skills、official docs 或受控 runtime evidence。
- 使用稳定的 source timestamp/fingerprint，不把每次 refresh 时间伪装成源更新时间。
- 对 ActivityLog、Task Atlas promotion 与 skill source 做上限和分块；保持 local-only 默认与显式 opt-out。

## P2 — 把 Eval 变成发布门槛

- 支持 top source/tool、必需 source path、禁止 top result 等 rank-aware assertion。
- 增加 negative lexical、UE 5.7/5.8 版本检索、空索引恢复、过期索引恢复与 source filter case。
- 同一 gate 覆盖 privacy 和 registry-schema parity。
- 任何 ranking、source、recommendation 变化之后都必须运行 `unreal.knowledge_eval_run`。

## UE 5.7 / 5.8 兼容批次

1. 只前移审查过的 JSON object compat shim、调用点/测试，以及 UE 5.8 unreachable-break 清理。
2. 根开发 host 改为 UE 5.7；UE 5.8 继续复用 UE 5.7 sample host。
3. `project_version_migration` 接受 UE 5.8；过渡期将 UE 5.6 标为 legacy/maintenance。
4. 两个优先引擎都通过后，再更新 installer、package name、CI/release gate 与安装文档。

## 发布门槛

- UE 5.7、UE 5.8 clean build 与完整 `UnrealMcp.*` automation 全部通过。
- 两个优先引擎都通过 endpoint smoke 和 MCP SDK conformance。
- 从干净本地状态运行 RAG 索引恢复与 rank-aware eval，全部通过。
- ToolRegistry 190/190，通过验证并与 plugin mirror 字节一致。
- v0.34 的 vetted tools、`call_tool`、Task Atlas、protocol shape 安全基线不退化。
- macOS 与 Windows 发布包都经过两个目标引擎验证后，文档才可从“计划”改写为“支持”。

## 首批明确不做

- 不整体合并 `experiment/v0.33-ue58-validation`。
- 不提交完整内容版 UE 5.8 sample project。
- 不引入 official-MCP Task Atlas generator，也不允许 policy bypass。
- 不立即删除 UE 5.6 编译兼容；EOL 另行基于验证证据决策。
