# v0.35 开发计划

语言：[[v0.35 Development Plan]] · [[v0.35 Development Plan CN]] · [[v0.35 Development Plan JP]]

v0.35 是一次过渡版本：将 UE 5.7 与 UE 5.8 确立为优先支持目标，并把本地 RAG 从“可用功能”提升为可靠的产品子系统。**Batch 1 已于 v0.35.0（2026-07-14）发布**：UE 5.7/5.8 主源码目标 + 知识索引可靠性与检索质量整改（release notes 见 `Docs/Release-2026-07b.md`，开发契约见 `Docs/Development-0.35.md`）。dual-variant 结构与可选的 Epic official-MCP 集成有意不纳入 Batch 1，作为待完成的 Batch 2（见文末）。

## 支持决策

| 引擎 | v0.35.0 结果 |
| --- | --- |
| UE 5.8 | 第一优先——已发布；Example57-host 干净 UBT 构建、RAG 17/17、Gate D 1/1、EngineCompat 2/2、migration 1/1，外加该引擎上的 v0.34 安全基线 |
| UE 5.7 | 第一优先——已发布；证据集与 UE 5.8 相同 |
| UE 5.6 | maintenance 编译 canary；不是 v0.35 的主发布门槛。v0.35 公开包面向 5.7/5.8；最后一个包含 5.6 打包的版本线是 v0.34.0 |

现有 UE 5.7 sample host（`Examples/UEvolveExample57`）同时用作 UE 5.8 的构建/commandlet host，没有新增数 GB 的完整 5.8 内容工程。v0.33 实验分支仍只是验证证据、不是合并基线：v0.35.0 只前移了经审查的小型 JSON object compat shim 与 UE 5.8 unreachable-code 清理。Epic 的 UE 5.8 official MCP 保持可选、默认关闭，并且不得绕过 UEAtelier 现有的 policy、vetting 与 audit 边界。

## P0 — 先让知识索引可信

已在 v0.35.0 交付。

1. Gate D 与 Knowledge eval fixtures 使用隔离的 `indexRoot` 目录，测试清理不可能抹掉用户索引。
2. 搜索/推荐/refresh 返回机器可读的 `missing | empty | stale | ready | corrupt` 状态；Chat 按该字段刷新，且每个面板会话在首次自动刷新失败后不再重试。
3. `cards.jsonl`/`index.json` 通过可恢复的 last-known-good `.bak` 对进行 staged/verified 替换；空卡 refresh 默认 fail-closed，写入被中断时下次加载自动恢复。`allowEmptyIndex` 硬性限制在 `Saved/UnrealMcp/Tests` 下的显式 `indexRoot`。未变化的热读取走 size/timestamp 快路径。
4. ToolRegistry schema 公开了 `includeActivityLog`、`includeSkills`、`sourceKinds`、`groupByKind`，以及新增的 `includePromotedSources` 与 `allowEmptyIndex`。
5. 索引 schema `UEvolve.KnowledgeIndex.v2` 记录 `buildId`、`generatedAtUtc`、`sourceFingerprint`、`cardsSha256`、卡片/来源/引擎计数、去重/截断计数与构建参数。
6. ActivityLog 索引为显式 opt-in（默认 false）；仅仅是 local-only 存储，不等于已经授权把 chat summary 注入 provider context。索引 containment 为 fail-closed，且不跟随 symlink/reparse point。

## P1 — 提升检索质量

已在 v0.35.0 交付。

- `5.7`、`5.8`、`UE5.7`、`UE5.8` 等引擎版本 token 在分词后保留；普通 `x.y` 数字 token 仅对已知引擎版本触发过滤，查询产品版本 `0.35` 或 Python `3.11` 不再隐藏带版本文档。
- Latin 词边界按 ASCII 定义、跨平台确定：`ui` 不再命中 `build`，且在 CJK 文本内可检索；CJK bigram 在所有平台上行为一致。
- 原始 query token 排名高于 synonym expansion（扩展项权重 0.55），不再等权。
- 全局截断在卡片上限之前保留 source-kind/引擎多样性，高权重文档不能挤掉 skills、official docs 或受控 runtime evidence；同分排序确定性。
- 卡片元数据区分 `sourceUpdatedAt`、`indexedAt` 与 `contentSha256`，不再把每次 refresh 时间伪装成源更新时间；official-doc 行与 KnowledgeCard 携带 `engineVersion`。
- ActivityLog、Task Atlas promotion 与 skill source 保持上限和分块，保留 local-only 默认与 ActivityLog 的显式 opt-in。（独立的 `ScoreToolForTask` 工具推荐匹配器的边界感知打分留作 Batch 2 fast-follow。）

## P2 — 把 Eval 变成发布门槛

已在 v0.35.0 交付。

- Eval schema v2（`UEvolve.KnowledgeEval.v2`）支持 rank-aware assertion：`expectSourcePathsAtK`、`expectAnySourcePathContains`、`forbidTopSourcePathContains`、`expectToolAtRank`，并汇总 rank-assertion 计数/通过率。
- 17 项 RAG automation 基线覆盖中断恢复、stale/corrupt append 拒绝、隔离空索引门控、已知引擎数字过滤、确定性 CJK/Latin 检索与原始 token 排名，在两个主引擎上执行。
- privacy 与 registry-schema parity 检查仍在同一 gate。
- 任何 ranking、source、recommendation 变化之后都必须运行 `unreal.knowledge_eval_run`（仍为长期规则；UE 5.8 实机运行：隔离索引上 8/8 case、3/3 rank assertion）。

## UE 5.7 / 5.8 兼容批次

已在 v0.35.0 交付。

1. 只前移了审查过的 JSON object compat shim、调用点/测试，以及 UE 5.8 unreachable-break 清理；所有引擎版本逻辑仍集中在 `UnrealMcpEngineCompat.h`，`Tools/check_ue56_compat.py` 新增 UE 5.8 `FJsonObject::Values` 模式。
2. 根开发 host 已改为 UE 5.7；UE 5.8 复用 UE 5.7 sample host。
3. `unreal.project_version_migration` 接受 `5.6`、`5.7`、`5.8`，并报告支持层级：primary 5.7/5.8、maintenance 5.6。
4. installer（带引擎支持门槛的 `Tools/install_unrealmcp_to_project.py`）、`ue57-ue58` 包名、CI/release gate（从精确 SHA 打 artifact-only 候选包、fail-closed 的 tag↔`VersionName` 校验）与安装文档均在两个主引擎通过后更新。
